# Prefetch Service Worker Setup Example

This example demonstrates how to manually integrate Prefetch Service Worker in a Next.js application after running the `prefetch-migrate` tool.

## What the migrate tool did

1. ✅ Detected Next.js framework
2. ✅ Installed `@norejs/prefetch` package
3. ✅ Created/updated `public/service-worker.js` with Prefetch integration
4. ✅ Provided registration code template

## What we did manually

1. ✅ Created `app/components/ServiceWorkerRegistration.js` - A client component for SW registration
2. ✅ Updated `app/layout.js` - Added the registration component to the root layout
3. ✅ Configured Prefetch settings - Set API matcher, cache settings, and debug mode

## Key Files

### `app/components/ServiceWorkerRegistration.js`
```javascript
'use client';

import { useEffect } from 'react';

export default function ServiceWorkerRegistration() {
  useEffect(() => {
    // Register Prefetch Service Worker
    if ('serviceWorker' in navigator) {
      navigator.serviceWorker.register('/service-worker.js')
        .then(registration => {
          console.log('Prefetch Service Worker registered:', registration);
          
          // Send initialization message to Service Worker
          if (registration.active) {
            registration.active.postMessage({
              type: 'PREFETCH_INIT',
              config: {
                apiMatcher: '/api/*',
                defaultExpireTime: 30000,
                maxCacheSize: 100,
                debug: true
              }
            });
          }
        })
        .catch(error => console.error('Prefetch Service Worker registration failed:', error));
    }
  }, []);

  return null;
}
```

### `app/layout.js`
```javascript
import './globals.css';
import ServiceWorkerRegistration from './components/ServiceWorkerRegistration';

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>
        {children}
        <ServiceWorkerRegistration />
      </body>
    </html>
  );
}
```

### `public/service-worker.js`
Generated by the migrate tool with Prefetch integration code.

## How to test

1. Start the development server:
   ```bash
   npm run dev
   ```

2. Open browser DevTools (F12)

3. Go to Application > Service Workers

4. Verify the Service Worker is registered

5. Check Console for registration messages

6. Test API requests to see caching in action

## Configuration Options

You can customize the Prefetch configuration in `ServiceWorkerRegistration.js`:

```javascript
config: {
  apiMatcher: '/api/*',        // Which URLs to cache
  defaultExpireTime: 30000,    // Cache expiration (ms)
  maxCacheSize: 100,           // Maximum cache entries
  debug: true                  // Enable debug logging
}
```

## Benefits of Manual Integration

✅ **Full Control**: You decide where and how to register the Service Worker
✅ **Framework Specific**: Use proper Next.js patterns (client components, useEffect)
✅ **Customizable**: Easy to modify configuration and behavior
✅ **Maintainable**: Clear separation of concerns
✅ **Testable**: Can be easily unit tested
